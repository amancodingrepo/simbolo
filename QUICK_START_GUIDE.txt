CRITICAL DETAILS TO PRESERVE
===========================================

1. EXACT CLASS STRUCTURE
   - Maintain all class names for CSS styling
   - Keep class order for specificity
   - Don't remove "pt-0", "flat-spacing", etc.

2. DATA ATTRIBUTES
   - data-clone="5" for carousels
   - data-speed="1500", data-to="120", data-inviewport="yes" for counters
   - These control functionality, not just styling

3. ANIMATION CLASSES
   - effectFade + direction (fadeUp, fadeRotateX, etc.)
   - Keep on exact elements
   - img-transform-3 on images
   - These must map to CSS keyframes

4. RESPONSIVE COLUMNS
   - col-xxl-7 col-lg-6, col-xxl-5 col-lg-6
   - col-md-6, col-lg-4 patterns
   - Different sections use different grids

5. BUTTON STYLES
   - tf-btn (primary - gradient background)
   - tf-btn-2 (secondary - outline style)
   - Distinguish between these two

6. TEXT GRADIENTS
   - text-gradient-1: Used on heroes and main titles
   - text-gradient-2: Used on section titles
   - These need distinct CSS definitions

7. COUNTER ANIMATIONS
   - Must trigger on scroll into viewport
   - Needs Intersection Observer implementation
   - Speed and target values from data attributes

8. CAROUSEL FUNCTIONALITY
   - Infinite loop with data-clone="5"
   - Smooth scroll
   - Responsive width

9. FORM STRUCTURE
   - Contact form must support:
     * Multiple input types (text, email, textarea)
     * Validation
     * Submit handling
     * Optional CAPTCHA or similar

10. MULTI-CONTAINER SECTIONS
    - ContactSection has 3 containers
    - FeaturesSection has multiple containers
    - Don't flatten these structures

===========================================
KEY GOTCHAS TO AVOID
===========================================

1. NOT preserving class names exactly
   - Don't simplify or consolidate classes
   - CSS styling depends on exact classes

2. REMOVING responsive column breakpoints
   - col-xxl-7 is not the same as col-lg-6
   - Different pages use different grids
   - Test on actual device sizes

3. FORGETTING data attributes
   - These control JavaScript behavior
   - data-clone, data-speed, data-to, data-inviewport
   - Loss of these breaks functionality

4. MISSING animation triggers
   - Not implementing Intersection Observer
   - effectFade animations won't trigger on scroll
   - Counters won't animate on view

5. FLATTENING multi-container sections
   - FeaturesSection uses 3 containers purposefully
   - ContactSection uses 3 containers
   - Maintaining structure is important

6. BUTTON STYLE CONFUSION
   - tf-btn ≠ tf-btn-2
   - Different colors/styles
   - Consistent throughout design

7. NOT TESTING CAROUSELS
   - data-clone must work for infinite loop
   - Mobile width changes affect carousel
   - Test with actual slide counts

8. FORM WITHOUT BACKEND
   - Contact form needs submission handler
   - Don't forget email/validation backend
   - Test form submission end-to-end

9. MISSING GRADIENT TEXT CSS
   - text-gradient-1 and text-gradient-2 need CSS
   - Can't just use text-white and expect gradients
   - Define via background-clip: text

10. NOT OPTIMIZING IMAGES
    - Large hero images need optimization
    - Partner logos should be SVGs
    - Next.js Image component recommended
